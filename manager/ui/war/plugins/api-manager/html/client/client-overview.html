Client Details

This is the Client details page. Use this page to modify the Client's meta-data, policies, and contracts. There is no need to follow the tabs in order, but note that you will need to fill out a minimum amount of data before the Client can be registered with the Gateway. In particular, the Client must have at least one API Contract (see the "Contracts" tab).

Contracts

The 'Contracts' tab is where you can manage all of the API Contracts for this Client. A API Contract is simply a link between this Client and a provided API the the Client consumes.

Policies

The 'Policies' tab allows you to manage the Client-level policies that should be applied whenever a request is made to any API consumed by this Client.

APIs

The 'APIs' tab lists all of the APIs consumed by this Client, including showing the API Key and live endpoint for each. Go here if you want to figure out how to invoke an API once you have a Contract for it.

Change Log

The 'Change Log' tab shows a history of all the changes made to the Client. Essentially it is an audit log.
