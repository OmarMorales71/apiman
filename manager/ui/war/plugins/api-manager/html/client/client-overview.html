Client Details

Use the client details to modify the client's metadata, policies and contracts. There is no need to follow the tabs in order, but you will need to fill out a minimum amount of data before the client can be registered with the gateway. In particular, the client must have at least one API contract (see tab Contracts).

Contracts

In the Contracts tab you can manage all API contracts for this client. An API contract is a link between this client and a provided API the client consumes.

Policies

The Policies tab allows you to manage the client-level policies that should be applied whenever a request is made to any API consumed by this client.

APIs

The APIs tab lists all APIs consumed by this client. Use the list to display also the API Key and live endpoint for each API. Use this tab to figure out how to invoke an API once you have created a contract for it.

Change Log

The Change Log tab shows a history of all the changes made to the client. Essentially this is an audit log.
